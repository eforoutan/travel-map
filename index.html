About

Cities I've Been

An interactive world map for pinning everywhere you've wandered. Search a city or right-click the map to drop a pin, label it, and publish it to share across devices. The recommender suggests popular US cities you haven't pinned yet.

Leaflet.js
Open-source web-mapping engine handling the map, markers, and pan/zoom. No API key needed.
Esri-style light canvas
CARTO Positron raster tiles over OpenStreetMap data — a clean neutral basemap.
OpenStreetMap · Nominatim
Free worldwide geocoding & reverse-geocoding for city search and map clicks.
US recommender
A curated list of popular US destinations, filtered to exclude anywhere near your existing pins.
Shared places.json
Pins load from a file in the repo so every device sees the same map; localStorage caches local edits.
GitHub Contents API
With a token in Settings, Publish commits directly to the repo; otherwise it downloads the file.

A single index.html — drop it on GitHub Pages and it just works.

Publish your map

File downloaded

A file called places.json just downloaded. To share these pins across devices, commit it to your repo:

  1. Open your repo on GitHub (the one with index.html).
  2. Click Add file → Upload files.
  3. Drag in places.json and allow it to replace the old one.
  4. Click Commit changes.

Tip: add a GitHub token in Settings and Publish will commit automatically — no manual upload.

Auto-publish setup

Publish directly to GitHub

Add a GitHub token and Publish will commit places.json automatically. Leave empty to keep using the download method.

Where your token lives

Stored only in this browser on this device — never written into the repo or the public page, so visitors can't see it. Treat it like a password.

Use a fine-grained token scoped to only this one repository, with Contents: Read and write permission. If your test passes but Publish fails with 403, the Contents permission is read-only — set it to Read and write.
